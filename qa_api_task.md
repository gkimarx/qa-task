# Тестовое задание для QA

Ниже описаны два взаимосвязанных API-метода микросервиса, отвечающего за создание и назначение задач в системе управления обращениями технической поддержки интернет-магазина. В таблицах и тексте есть общая информация, структура тела запроса и ответа и прочее.

Просмотри содержание документа и предложи список проверок, которые считаешь необходимыми. 

---

# 1. API Tasks — создание задачи

## Общая информация

| **HTTP-метод** | POST |
| --- | --- |
| **URL** | https://helpdesk.company.com/tasks/v1/task/create |
| **Описание** | Создание задач — карточек обращений пользователей, которые сотрудники техподдержки создают вручную или автоматически при поступлении запроса. Задача фиксирует суть проблемы, срок и ответственный отдел |
| **Требования** | Поле `title` должно принимать не более 10 символов. Поле `priority` должно содержать одно из значений: "low", "medium", "high". |

## 📤 Описание запроса

### Параметры тела запроса

|  | **Наименование** | **Код** | **Тип** | **Обязателен** | **Комментарий/Пример** |
| --- | --- | --- | --- | --- | --- |
| 1 | Заголовок задачи | title | string | Да | Поле БД tasks.tasks.title |
| 2 | Описание задачи | description | string | Нет | Поле БД tasks.tasks.description |
| 3 | Приоритет | priority | string | Да | Допустимые значения: "low", "medium", "high". Поле БД tasks.tasks.priority |
| 4 | Срок выполнения | deadline | string | Да | Дата передаётся в формате ISO 8601. **Пример**: "2023-08-08T15:26:48.000Z". Поле БД tasks.tasks.deadline |
| 6 | Идентификатор отдела | departmentId | number | Да | Поле БД tasks.tasks.department_id |

### Пример запроса

```json
POST https://helpdesk.company.com/tasks/v1/task/create

{
  "title": "Fix login bug",
  "description": "Users cannot log in after the last update",
  "priority": "High",
  "deadline": "2023-09-01 12:00:00",
  "departmentId": 42
}
```

## 📥 Описание ответа

### Параметры тела ответа

|  | **Наименование** | **Код** | **Тип** | **Обязателен** | **Комментарий/Пример** |
| --- | --- | --- | --- | --- | --- |
| 1 | Флаг успешности запроса | success | boolean | Да | |
| 2 | Причина | reason | string | Нет | |
| 3 | Идентификатор задачи | id | number | Да | Поле БД tasks.tasks.id |
| 4 | Заголовок задачи | title | string | Да | Поле БД tasks.tasks.title |
| 5 | Описание задачи | description | string | Нет | Поле БД tasks.tasks.description |
| 6 | Приоритет | priority | string | Да | Поле БД tasks.tasks.priority |
| 7 | Статус задачи | status | string | Да | Начальный статус при создании: "new". Поле БД tasks.tasks.status |
| 8 | Дата создания задачи | createdAt | string | Да | Поле БД tasks.tasks.created_at |
| 9 | Срок выполнения | deadline | string | Да | Дата передаётся в формате ISO 8601. **Пример**: "2023-08-08T15:26:48.000Z". Поле БД tasks.tasks.deadline |
| 10 | Идентификатор отдела | departmentId | number | Да | Поле БД tasks.tasks.department_id |

### Пример успешного ответа

```json
{
  "success": true,
  "id": 567,
  "title": "Fix login bug",
  "description": "Users cannot log in after the last update",
  "priority": "High",
  "status": "new",
  "createdAt": "2023-08-15 10:30:00",
  "deadline": "2023-09-01 12:00:00",
  "departmentId": 42
}
```

### Пример неуспешного ответа

```json
{ "success": false, "reason": "internal error" }
```

# 2. API Tasks — назначение исполнителя

## Общая информация

| **HTTP-метод** | POST |
| --- | --- |
| **URL** | https://helpdesk.company.com/tasks/v1/task/assign |
| **Описание** | Назначение сотрудника исполнителем задачи. Назначить исполнителя можно только на задачу со статусом "new" |

## 📤 Описание запроса

### Параметры тела запроса

| **№** | **Наименование** | **Код** | **Тип** | **Обязателен** | **Комментарий/Пример** |
| --- | --- | --- | --- | --- | --- |
| 1 | Идентификатор задачи | taskId | number | Да | |
| 2 | Идентификатор исполнителя | assigneeId | string | Да | UUID сотрудника |

### Пример запроса

```json
POST https://helpdesk.company.com/tasks/v1/task/assign

{
  "taskId": 567,
  "assignee": {
    "id": "a3f2d1c0-1234-5678-abcd-ef0123456789"
  }
}
```

## 📥 Описание ответа

### Параметры тела ответа

| **№** | **Наименование** | **Код** | **Тип** | **Обязателен** | **Комментарий/Пример** |
| --- | --- | --- | --- | --- | --- |
| 1 | Флаг успешности назначения | success | boolean | Да | |
| 2 | Код ошибки | error | string | Нет | Возвращается в случае ошибки назначения |

### Пример успешного ответа

```json
{ "success": true }
```

### Пример неуспешного ответа

```json
{
  "success": false,
  "error": "alreadyAssigned"
}
```
